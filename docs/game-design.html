SYSTEM DESIGN
クラス設計と依存関係
責務の分離
  • Game
    • SceneManager / TimeManager / AudioManager に依存
  • SceneManager
    • 保持：TitleScene / GameScene / ResultScene
    • 各シーンは update, render を実装
  • GameScene
    • 管理：Player, EnemyManager, SpawnManager, ItemManager
    • 参照：TimeManager, ExperienceSystem, UIManager
  • EnemyManager
    • Spawner に敵作成を依頼（Factory）
    • 敵共通インタフェース IEnemy によるポリモーフィズム
  • ExperienceSystem
    • イベント onLevelUp を Player に通知
    • 敵撃破時に Enemy → ExperienceSystem が経験値付与

GAME DESIGN
ゲーム画面のState遷移
Scene + GameState
  • Scene遷移
    1. TitleScene：キャラ選択 → 「Start」 → GameScene
    2. GameScene：規定時間後ボス出現 → 撃破で ResultScene(Clear)、死亡で ResultScene(Dead)
    3. ResultScene：リトライ or タイトルへ
  • GameScene内のGameState
    • Playing_Normal：通常敵 / エリート湧き
    • Transition_To_Boss：5分経過 → 敵湧き停止 → ボス演出
    • Boss_Battle：ボス＋雑魚制御
    • Clear：ボス撃破演出 → Resultへ
    • GameOver：死亡 → Resultへ

Stateパターン or 単純な enum + switch で実装可能。

GAME DESIGN
プレイヤーのState遷移
アクションと無敵時間
  • 基本State
    • Idle：待機
    • Run：左右移動
    • Jump：ジャンプ / 落下
    • Attack：近接 / 遠距離攻撃
    • Dash：回避行動（短時間無敵）
    • Hit：被弾リアクション
    • Dead：死亡、操作不可
  • 代表的な遷移
    • Idle ⇄ Run（移動入力）
    • Run/Idle → Jump（ジャンプボタン）
    • Run/Idle/Jump → Attack（攻撃ボタン）
    • 任意State → Dash（ダッシュボタン、クールダウン付き）
    • 非 Dead State → Hit（被弾時／無敵でなければ）
    • HP<=0 → Dead → GameState:GameOver

ENEMY DESIGN
敵の種類とスポーン方法
通常 / エリート / ボス
敵種別
  • 近接：接近して殴る、HP中、速度中
  • 遠距離：射撃、HP低、射程長
  • 自爆：一定距離で自爆、範囲ダメージ
  • ゾーナ：フィールドに設置型攻撃（トラップ、持続範囲）
  • エリート：上記の強化版、専用スキル持ち、アイテムバッグドロップ
スポーン設計
  • 左右画面外のスポーンポイントから出現
  • 通常敵：
    • 時間帯ごとに「敵テーブル」切り替え
    • 例：0～2分は近接メイン、2～4分は遠距離混ざる…
  • エリート：
    • 毎分（60秒ごと）に1体出現
    • 画面外ランダム、プレイヤー位置から距離を離してスポーン
  • ボス：
    • 5分経過で通常湧き停止 → ボス専用演出後スポーン

ENEMY DESIGN
敵のState遷移
AIの基本パターン
  • 共通State
    • Spawn：出現演出、短時間待機
    • Chase：プレイヤー方向へ移動
    • Attack：近接 / 射撃 / 自爆 / ゾーン設置
    • Stagger：ひるみ（オプション）
    • Dead：死亡、ドロップ処理
  • 自爆タイプ
    • 距離 < R1：Chase
    • 距離 < R2：Charge（点滅→自爆カウント）
    • タイマー0：Explode → Dead
  • ゾーナタイプ
    • 一定距離キープ → PlaceZone → クールダウン → Chase or 後退
  • エリート
    • HP閾値でフェーズ変化（移動速度UP、攻撃パターン追加など）
    • 死亡時：ItemBag のドロップ処理

PROGRESSION
アイテム / 宝箱 / 経験値
ドロップとレベルデザイン
  • アイテムバッグ
    • ドロップ元：エリートのみ
    • 内容物：ランダム 3～5 個のアイテム
    • 例：ステータス増加、パッシブ効果、回復、追加武器など
  • 宝箱（Chest）
    • マップにランダム配置（開始時にいくつか確定配置）
    • 中身：武器 1 つ（レアリティ / タイプを抽選）
    • 鍵 or 条件（特定敵撃破数など）で解錠も検討可
  • 経験値とレベル
    • 敵撃破 → 経験値オーブドロップ or 即時加算
    • レベルアップで「アイテム抽選」「ステータスアップ」
    • 必要経験値：指数的に上昇（例：EXP(n) = 20 * n^2 + 30）
    • レベル上限はテストで調整（例：Lv 30 前後）

PROGRESSION
時間経過による敵強化
スケーリングパラメータ
  • TimeManager が「経過秒数 t」を管理
  • 敵パラメータスケール例（t：分）
    • HP：HP(t) = HP_base * (1 + 0.25 * t)
    • 攻撃力：ATK(t) = ATK_base * (1 + 0.18 * t)
    • 移動速度：SPD(t) = SPD_base * (1 + 0.05 * t)
  • 経験値の上昇
    • ドロップ経験値：EXP_drop(t) = EXP_base * (1 + 0.2 * t)
    • 終盤でもレベルアップが気持ちよく起きるよう調整
  • 難易度ブレイクポイント
    • 2分、4分などで出現敵テーブルを段階的に切り替える
    • エリートの出現数上限を時間で増加させる

BOSS DESIGN
ボスのデザイン
フェーズ制 & テレグラフ
  • 役割
    • 5分間の積み上げ（ビルド）が試される DPS チェック
    • 横スクロールならではの画面端制圧 / 弾幕 / 範囲攻撃
  • フェーズ例
    • P1：近接突進 + 単純な飛び道具
    • P2（HP 70%以下）：床範囲攻撃（ゾーナ要素）、雑魚召喚
    • P3（HP 35%以下）：攻撃頻度アップ、攻撃パターン高速化
  • テレグラフ（予兆演出）
    • 予備動作アニメ、地面の予兆エリア、警告SE
    • 「覚えゲー」になりすぎないよう、視認しやすい演出
  • 報酬
    • ユニークアイテム or 新キャラ解放に紐づけるとリプレイ性UP

FLOW
ボス撃破演出 & リザルト画面
クリア / 死亡の分岐
  • ボス撃破後の演出
    • スローモーション + ヒットストップ
    • カメラズームイン → ズームアウト
    • ボス爆散 or 崩れ落ちアニメーション
    • BGMフェードアウト → 勝利ジングル再生
    • 取得アイテム/ステータスのハイライト表示後、フェード → リザルト
  • リザルト画面（共通）
    • プレイタイム、到達Wave/時間、撃破数（通常 / エリート / ボス）
    • 最終レベル、取得した主なアイテム / 武器
    • スコア（クリアボーナス / ボス撃破ボーナスなど）
  • Clearパターン
    • 「MISSION CLEAR」演出
    • 次の解放要素があればここで表示
  • 死亡パターン
    • 死亡直前のリプレイ風スロー + 赤いフェード
    • 「原因」（どの敵/攻撃でやられたか）を表示すると学習しやすい

AUDIO DESIGN
音響戦略（CV / SE / BGM）
テンポ感とフィードバック
  • BGM
    • 通常時：テンポ中程度、ループしやすいトラック
    • エリート出現：短いイントロSE + BGMにレイヤー追加（緊張感アップ）
    • ボス戦：専用曲。イントロ → ループ部 → ラスト撃破ジングル
    • リザルト：短めで耳に残るフレーズ（Clear / Dead で別曲）
  • SE
    • 攻撃・被弾・回避に明確な音量・ピッチ差をつける
    • レベルアップ時は気持ちいい上昇系SE
    • ボスの大技には事前警告SE（チャージ音）
  • CV（ボイス）
    • プレイヤー：攻撃 / 被弾 / ダッシュ / レベルアップの短いボイス
    • ボス：フェーズ移行時の一言ボイスでキャラ性付与
    • リザルト：Clear / Dead でそれぞれのボイスを再生
  • 実装
    • AudioManager にカテゴリ別ボリューム（BGM / SE / VOICE）
    • 同時再生数制限、優先度制御で耳障りを防ぐ

TEST STRATEGY
テスト方針
機能・バランス・体感
  • 機能テスト
    • シーン遷移：タイトル → ゲーム → リザルト（Clear / Dead）
    • State遷移：プレイヤー / 敵の全パターン確認
    • スポーン：通常 / エリート / ボスの出現タイミング
    • ドロップ：エリートのみアイテムバッグを落とすか
    • 宝箱：マップランダム配置、武器が必ず1つ入るか
  • バランステスト
    • 時間ごとの敵強さ・経験値上昇が妥当か
    • 想定プレイ（5分）で何レベル程度になるか
    • ビルド差：強ビルド / 弱ビルドでクリア率を確認
  • 体感テスト（プレイテスト）
    • 1分ごとのエリート出現がテンポ良く感じるか
    • ボス戦が理不尽ではなく「学べる難しさ」か
    • SE / BGM の音量バランス、長時間でも疲れないか
  • 自動テスト候補
    • レベルテーブルの単体テスト（経験値計算）
    • 敵スケーリング関数の境界値テスト
    • スポーン間隔 / 上限数の検証

このスライドをベースに、各機能ごとの詳細仕様書やタスク分解を行う想定。
